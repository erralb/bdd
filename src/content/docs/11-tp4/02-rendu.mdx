## Fichiers à rendre

Les deux membres du binôme doivent déposer une **archive ZIP** sur Moodle contenant les fichiers suivants :

1. Un fichier `0-README.md` contenant les noms et prénoms des membres du groupe et des remarques éventuelles
2. Un fichier `1-clinique-normalisation.md` contenant le schéma en 3NF (Exercice 1)
3. Un fichier `2-clinique-ddl.sql` contenant le code SQL de création des tables (Exercice 2)
4. Un fichier `3-1-clinique-algebre.md` contenant les requêtes d'algèbre relationnelle (Exercice 3.1)
5. Un fichier `3-2-clinique-dql.sql` contenant les requêtes SQL (Exercice 3.2)
6. Un fichier `4-clinique-dml.sql` contenant les requêtes SQL pour les actions DML (Exercice 4)
7. Un fichier `5-clinique-evolution.sql` contenant le code SQL pour la table `FACTURE` (Exercice 5)
8. Un fichier `6-clinique-triggers.sql` contenant les triggers, procédures stockées et vues (Exercice 6)
9. Votre fichier de base de données SQLite `clinique.sqlite` sur lequel vous avez travaillé
10. Un export de votre base de données au format SQL nommé `clinique.sql`
    Vous pouvez utiliser la commande suivante depuis un terminal pour obtenir cet export :
    ```bash
    sqlite3 clinique.sqlite .dump > clinique-export.sql
    ```

## Archive ZIP

- L'archive ZIP doit être nommée `clinique-NOM1-NOM2.zip` (en remplaçant `NOM1` et `NOM2` par vos noms de famille respectifs).
- Votre archive ZIP doit être déposée sur Moodle avant la date limite de remise : **Mardi 3 décembre à 23h59**.
- Les fichiers doivent être nommés correctement et respecter les formats demandés.
- Les fichiers doivent être correctement formatés, lisibles, sans erreurs de syntaxe.
- Les différentes requêtes doivent être clairement identifiées dans les fichiers, notamment en étant précédées de commentaires
- merci de ne pas inclure de fichiers inutiles dans votre archive.

## Évaluation

Votre projet sera évalué selon les critères suivants :

- Respect des consignes, notamment les formats et noms des fichiers demandés
- Qualité de la décomposition en 3NF
- Qualité du code SQL, notamment la création des tables et les contraintes ainsi que les commentaires dans les fichiers

Pour corriger, je vais partir d'une copie du fichier [clinique-non-normalisee.sqlite](/bdd/sample/clinique-non-normalisee.sqlite), puis appliquer successivement le code de vos fichiers SQL.

Avant de rendre votre travail, vous pouvez tester vos fichiers en les appliquant sur une copie de la base de données fournie.

Si vous avez des questions, n'hésitez pas à les poser sur le forum Moodle ou par mail.
